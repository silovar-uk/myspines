# ADR 0001: 専用編集エンジンの前に操作の縦切りを作る

## Status

Accepted for `0.1.x`.

## Context

myspinesで最も不確実なのは、ライブラリ選定ではなく、日本語を打ち、分割し、構造を動かし、元の文章へ戻る一連の手触り。

リポジトリは空であり、最初からProseMirrorを導入すると、スキーマ・NodeView・トランザクション設計と、プロダクトの操作検証が同時進行になる。

## Decision

最初の公開版は依存パッケージなしで作り、WRITE / SHAPE、階層操作、Undo、保存、コピーの一連を実原稿で検証できる状態にする。

専用編集エンジンは、IMEと選択復元の実機記録を得てから導入判断する。

## Why not

現在方式を最終編集基盤と決めたわけではない。複数ノードをまたぐ選択、長期履歴、協調編集、複雑な変換には専用エンジンが有利。

ただし、最初の公開前にそれらを実装すると、myspines固有の操作感より編集エンジン都合が先に固まるため採用しない。
