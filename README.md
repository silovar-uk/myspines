# myspines

> 段落を動かしながら、長い文章を書くためのローカルファーストエディタ。

myspinesは、本文を書きながらアウトラインを整理し、必要な範囲をAIへ渡せる長文執筆ツールです。
UI v3では、アウトラインを「地図」、本文を「主作業領域」として明確に分けています。

## 現在の実装

- **本文 / 構成 / メモ**
  - 本文画面は、可変幅アウトラインと連続した原稿面のSplit View
  - 構成画面は、見出し・本文冒頭・文字数を比較する表形式
  - メモには、原稿全体・現在の項目・次に書くことを保存
- **端末別レイアウト**
  - デスクトップ：アウトライン幅を220〜400pxで変更可能
  - タブレット：アウトラインを左から重ねて表示
  - スマホ：本文・構成・メモの三画面と段落操作シート
- **階層編集**
  - `Enter`で段落分割
  - `Shift + Enter`で段落内改行
  - `Tab / Shift + Tab`で階層変更
  - `Alt + Shift + ↑ / ↓`で上下移動
- **集中表示と折り畳み**
- **未配置**
  - まだ本文内の居場所を決めない断片を退避
- **AIとの往復**
  - 選択範囲、段落、部分木、本全体をコピー
  - 本文のみ、見出し付き、Markdown、AI用の形式
- **取り込みと書き出し**
  - プレーンテキスト、Markdown、myspines JSON
  - Markdown、テキスト、完全JSONの書き出し
- **端末内保存**
  - IndexedDBへ自動保存
  - アカウント、クラウド同期、外部送信なし
- **PWA**
  - アプリシェルをオフライン利用可能

## 使い始める

GitHub Pages:

`https://silovar-uk.github.io/myspines/`

ローカルでは、静的ファイルサーバーでリポジトリ直下を開きます。

```bash
python -m http.server 4173
```

その後、`http://localhost:4173`を開きます。

## ショートカット

| 操作 | キー |
|---|---|
| 段落を分割 | `Enter` |
| 段落内で改行 | `Shift + Enter` |
| 一段深くする | `Tab` |
| 一段浅くする | `Shift + Tab` |
| 上下へ移動 | `Alt + Shift + ↑ / ↓` |
| 本文 / 構成 | `Ctrl / Cmd + Shift + O` |
| メモ | `Ctrl / Cmd + Shift + M` |
| 形式を選んでコピー | `Ctrl / Cmd + Alt + C` |
| 集中 / 一つ上へ戻る | `Ctrl / Cmd + . / ,` |
| Undo / Redo | `Ctrl / Cmd + Z / Shift + Z` |

## 保存について

原稿はブラウザのIndexedDBへ保存されます。myspinesが本文をサーバーやAIへ送信することはありません。

ブラウザデータを削除すると原稿も失われる可能性があります。節目ごとに「完全JSONを退避する」を利用してください。

## UI v3の原則

- 主役は本文。アウトラインは本文を支える地図として扱う
- タイトル、現在地、本文の左端を同じ軸へ揃える
- 本文全体をカードで囲まず、38〜40字の連続した原稿面にする
- アウトラインと本文の間は、隙間ではなく一本の境界線で分ける
- アウトラインの文字数は、最上位と選択項目だけを初期表示する
- 構成画面はナビゲーションではなく、比較と再配置のための表にする
- 段落操作はPCでは左ガター、スマホでは下部シートに置く
- 1040px未満では本文幅を優先し、アウトラインをオーバーレイ化する

## 技術方針

現在の`0.3.x`は、実原稿で操作感を検証できる依存パッケージなしの縦切りMVPです。

- 編集意図は`beforeinput`を中心に扱う
- IME変換中は構造操作を起こさない
- 構造操作は一回のUndoで戻す
- 原稿データを自動で正規化しない
- 保存失敗だけを明確に伝える

専用編集エンジンへの移行は、IME・選択・Undo・モバイル入力の実機検証結果を基に判断します。
