# myspines

> 段落を動かしながら、長い文章を書くためのローカルファーストエディタ。

myspinesは、本文を書きながらアウトラインを整理し、必要な範囲をAIへ渡せる長文執筆ツールです。
UI v2では、kakidasの明快な情報設計を基準に、操作の見つけやすさと入力欄への近さを優先しています。

## 現在の実装

- **本文 / 構成 / メモ**
  - 本文画面では、開閉式アウトラインを確認しながら執筆
  - 構成画面では、階層・順番・文字量を整理
  - メモには、原稿全体・現在の項目・次に書くことを保存
- **階層編集**
  - `Enter`で段落分割
  - `Shift + Enter`で段落内改行
  - `Tab / Shift + Tab`で階層変更
  - `Alt + Shift + ↑ / ↓`で上下移動
- **集中表示と折り畳み**
- **未配置**
  - まだ本文内の居場所を決めない断片を退避
- **AIとの往復**
  - 選択範囲、段落、部分木、本全体をコピー
  - 本文のみ、見出し付き、Markdown、AI用の形式
- **取り込みと書き出し**
  - プレーンテキスト、Markdown、myspines JSON
  - Markdown、テキスト、完全JSONの書き出し
- **端末内保存**
  - IndexedDBへ自動保存
  - アカウント、クラウド同期、外部送信なし
- **PWA**
  - アプリシェルをオフライン利用可能

## 使い始める

GitHub Pages:

`https://silovar-uk.github.io/myspines/`

ローカルでは、静的ファイルサーバーでリポジトリ直下を開きます。

```bash
python -m http.server 4173
```

その後、`http://localhost:4173`を開きます。

## ショートカット

| 操作 | キー |
|---|---|
| 段落を分割 | `Enter` |
| 段落内で改行 | `Shift + Enter` |
| 一段深くする | `Tab` |
| 一段浅くする | `Shift + Tab` |
| 上下へ移動 | `Alt + Shift + ↑ / ↓` |
| 本文 / 構成 | `Ctrl / Cmd + Shift + O` |
| メモ | `Ctrl / Cmd + Shift + M` |
| 形式を選んでコピー | `Ctrl / Cmd + Alt + C` |
| 集中 / 一つ上へ戻る | `Ctrl / Cmd + . / ,` |
| Undo / Redo | `Ctrl / Cmd + Z / Shift + Z` |

## 保存について

原稿はブラウザのIndexedDBへ保存されます。myspinesが本文をサーバーやAIへ送信することはありません。

ブラウザデータを削除すると原稿も失われる可能性があります。節目ごとに「完全JSONを退避する」を利用してください。

## UI v2の方針

- 思想を表面の専門用語として見せず、操作の予測可能性として実装する
- `WRITE / SHAPE / Spine / Margin`を主要UIから外し、`本文 / 構成 / アウトライン / メモ`へ統一
- 見出しのない段落では、空の見出し欄を常時表示しない
- 本文画面では、アウトラインと原稿を同時に確認できる
- スマホの主要ナビは、本文・構成・メモの三つだけにする
- 書き出しを右上の主要操作として見つけやすくする

## 技術方針

現在の`0.2.x`は、実原稿で操作感を検証できる依存パッケージなしの縦切りMVPです。

- 編集意図は`beforeinput`を中心に扱う
- IME変換中は構造操作を起こさない
- 構造操作は一回のUndoで戻す
- 原稿データを自動で正規化しない
- 保存失敗だけを明確に伝える

専用編集エンジンへの移行は、IME・選択・Undo・モバイル入力の実機検証結果を基に判断します。
